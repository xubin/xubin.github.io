redux

store

  • 每一个应用只有一个store 通过createStore(fn) 创建store 并返回新的store

  • store 包含所有的对象数据

createStore

state

store 包含所有的对象数据 如果要获取某一个时间点数据 就要对 store 生成快照 这种某一个时间点的数据叫state 当前时刻的state 通过 store.getState() 获取

state 更react state 完全是俩回事 不能混淆

redux 规定一个state 对应一个 view state=>view view=>state

Action

state变化必然导致view变化 用户无法接触到state 只能接触到view 所以 state变化必须是view引起的 action就是view发出的通知（） 表示 state发生变化了 aciton 是一个对象 必须包含 type type表示action的名称

action规范参考

action creator

view 发多少种消息 就有多少种action

store.dispatch()

store.dispatch()是view发出action 的唯一方法

reducer

store 接收到action 必须给出一个 state 这样view就会发生变化，这种state的计算过程就是reducer

reducer 就是就是一个函数 返回新的state

Reducer 函数最重要的特征是，它是一个纯函数。也就是说，只要是同样的输入，必定得到同样的输出。

纯函数是函数式编程的概念，必须遵守以下一些约束

  • 不得改写参数

  • 不能调用系统 I/O 的API

  • 不能调用Date.now()或者Math.random()等不纯的方法，因为每次会得到不一样的结果

实际应用种 store.dispatch() 会触发 reducer 的自动执行 为此 store需要知道reducer函数 做法就是reducer出入 createStore函数
